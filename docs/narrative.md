# LAST MISSION — 내러티브 & 시스템 방향

> 2026-04-12 기준 설계 메모. 구현 단계에서 이 문서를 레퍼런스로 삼는다.

---

## 1. 한 줄 요약

이계종이 침범한 세계의 끝에서, 평생 호위 안드로이드가 어릴 적부터 지켜온 도련님을 *가문이 미리 마련해둔 지하 벙커*까지 호송하는 슬더스라이크. 매 거점, 매 임시수리는 미래의 자신에 대한 채무. 시작이 가장 강하고 끝이 가장 약하다.

---

## 2. 세계관

### 침공
- 미상의 시점에 **차원 균열**이 열리고 **이계종 / 외계 괴물**들이 침입
- 인류는 거의 멸종 직전. 도시도 군대도 없음
- 살아남은 자들은 흩어진 *소규모 잔존 그룹*으로 존재
- 다른 집단의 존재는 *언급만 됨, 등장 안 함*. 의도적으로 게임 안에서 외부 세계의 정보를 절제

### 주인공 일행이 속한 집단
- 한때 조직화된 단체였음. 간부 가문이 존재했음
- 게임 시작 시점에는 사실상 와해. 집단의 잔존 인원은 거의 없음
- 간부 가문이 미리 마련해둔 지하 벙커 = **"낙원"**
- 낙원의 정체: 사적 자금으로 구축한 가문 전용 셸터. 정부 시설 X. 한정된 공간, 한정된 자원, 한정된 인원

### 적 = 이계종
- 곤충형 / 키틴질 + 차원 균열에서 침입한 비현실 형태의 결합
- 군집 + 여왕 / 모체 구조 (해석 여지)
- *기계 적은 등장하지 않음*. 모든 적은 생물(또는 비생물 외계 존재)

### 로봇의 위치
- 인류가 만든 마지막 군용 기술의 산물 중 하나
- 사실상 *살아남은 마지막 군용 안드로이드*에 가까움. "라스트 미션"의 또 다른 의미

## 3. 캐릭터

### 주인공 — 군용 호위 안드로이드
- 외형: 현재 일러스트 (검은 머리 / 시안 액센트 / 흰 기체 슈트)
- 성격:
  - 거의 사람 같음. 학습형 AI, 일상 대화 가능, 감정 시뮬레이션 가능
  - 그러나 *마지막 1%가 닫혀있는* 존재. 로맨틱한 의미의 단어("아내")를 자기 자신에게 적용하지 못함
  - 임무 지향. 명령은 문자 그대로 해석
  - "나의 수호기사님"이라는 호칭의 무게를 *프로토콜로는 알지만 감정으로는 디코드 못 함*
- 기능: 가문 전속 *개인 경호* 안드로이드. 일반 솔저가 아닌 정예 1유닛
- 시작 상태: 최강. 완전체. 평생 단련된 기체
- 진행에 따라: 점점 부서짐. 임시수리로 땜질하며 버팀

### 주인 — 도련님 (소년, 17~19세)
- **화면에 등장하지 않음**. 대사 / 무선 통신으로만 존재
- 엔딩 씬에서 처음 시각화. 플레이어와 로봇이 *동시에* 처음 본다
- 정체: 와해된 집단의 마지막 간부 가문의 후계
- 부모(간부)의 행방: **사망/실종**. 게임 시작 시점에 이미 부재
- 평생을 안드로이드와 함께 살아옴. 어릴 때 부모가 붙여준 호위
- "수호기사"라는 호칭은 *어릴 적부터 안드로이드를 부르던 동화적 별명*. 어른이 되어가는 지금도 유지
- 로봇의 둔함을 알면서도 사랑함. 천천히 알려주려는 인내가 있음

### 둘의 관계 길이
- *평생*. 소년이 어릴 때부터 안드로이드가 곁에 있었음
- 수많은 작전 / 일상 / 보호의 시간을 함께 보냄
- 따라서 "아내" 발언은 충동이 아닌 *오랜 감정의 표출*

## 4. 핵심 주제

1. **"설령 이 몸이 부서져도"**
   - 로봇의 서약. 부서짐을 감수하며 임무 완수
   - 엔딩에서 의미가 재해석됨: *로봇이 주인의 사랑을 끝내 디코드하지 못했지만 그래도 끝까지 지키려 했다*

2. **이해하지 못해도 지킨다 / 이해하지 못해도 사랑한다**
   - 로봇은 주인의 감정 언어를 끝까지 디코드하지 못함
   - 주인은 그 둔함조차 *매력*으로 수용
   - "천천히 알려줄게" — 사랑의 언어가 지금 당장 양방향으로 성립하지 않아도 관계는 성립한다

3. **소모 / 퇴화의 곡선**
   - 전형적 로그라이크의 "성장"이 아니라 *잃어가는* 체험
   - 낙원 도착 = *덜 잃은 채로* 도착하는 것
   - 임시수리는 미래의 자신에게서 빌려오는 부채

4. **여정의 원형성**
   - 1막 보스와 같은 종이 최종 보스로 등장
   - 시작점에서 죽인 적의 *동족이 평생을 추적해 옴*
   - 가장 강할 때 마주친 적을 가장 약할 때 다시 마주함

## 5. 엔딩 구조

### 루트 A — 생환 엔딩 (해피)
- 최종 보스 정면 돌파 + 로봇 생존
- 로봇이 도련님 품에 도달
- 대사:
  > 도련님: "이제 나와 나의 아내를 위해 마련한 이곳에서, 인류 따위 어찌되든 살 수 있어."
  >
  > 로봇: "아내분도 잘 모시겠습니다만... 어기 계시죠?"
  >
  > 도련님 (웃으며): "...눈치없구나. 뭐, 그것도 네 매력이니까. 천천히 알려줄게, 나의 수호기사님."

**해석**:
- 도련님은 로봇을 *자신의 아내*로 여기고 있음. 미래 — 어른이 되는 동안 천천히 알려주겠다는 약속
- 로봇은 "아내"를 제3자로 해석, 주변을 둘러보며 찾는다 (물리적 부재 대상 탐색 프로토콜)
- 도련님은 실망하지 않음. 평생 알아온 이 안드로이드의 둔함을 사랑한 것
- "수호기사"는 어릴 적 별명에서 시작된 호칭. 군용 무기가 *기사*로 승격된 채로 평생 유지된 것

### 루트 B — 자폭 엔딩 (비극적 승리)
- 최종 보스에서 자폭 필살기로 즉시 승리, 로봇 소멸
- 도련님은 낙원에 도달하지만 혼자
- 로봇의 마지막 로그: 건조한 한 줄 — "임무 완수. 대상 생존 확인."
- 도련님은 평생을 함께한 수호기사에게 *고백할 기회*를 영영 잃음
- "천천히 알려줄게"라는 말을 할 대상이 사라짐

### 두 루트의 대비
- 루트 A: 서약을 지키고 *미래*를 얻음. 평생을 함께한 둘이 *드디어* 자신들만의 시간을 시작
- 루트 B: 서약을 지키고 *자신*을 잃음. 도련님은 호위 없는 첫 밤을 맞이
- 똑같은 "설령 이 몸이 부서져도"가 엔딩마다 다른 무게

## 6. 시스템 — Slay the Spire의 거울상

### 핵심 전환: 축적 → 소모

기존 로그라이크 덱빌더: 시작이 약하고 점점 강해짐 → 게임오버 or 클리어
**LAST MISSION**: 시작이 가장 강하고 점점 약해짐 → 낙원 도달 or 소멸

### 카드 베이스로 전환

기존 주사위/콤보 시스템 → **카드 베이스 (StS-like)** 로 재구성

- 매 턴 N장 드로우, 코스트 소비, 카드 효과 발동
- 시작 시 강력한 카드 다수 + 큰 덱
- 진행하면서 카드 제거, 등급 하락, 약화

(기존 주사위 시스템은 폐기. 코드는 일부 골격만 살리고 전투 로직은 새로 작성)

### HP 구조

**Max HP**: 층수에 따라 *영구 감소*. 수리 불가능. 기체 구조 손상
**Current HP**: Max HP 안에서 임시수리로 회복 가능. 부품 교체 / 땜질

"회복해도 뼈는 계속 금이 가는" 느낌. 후반으로 갈수록 Max HP 자체가 줄어 회복의 효과도 축소.

### "회복" 삭제 → "임시수리"

- 이전의 `HEL (힐)` 주사위: **제거**
- 이전의 "체력 회복 (HP 전체 회복)" 보상: **제거**
- 대신 "임시수리 (현재 HP +N, Max HP는 그대로)" 거점 행동
- 명칭은 항상 "임시수리". 플레이어가 "회복"이라는 단어를 보지 못하게

### 시작 상태

완전체:
- 강력한 카드 다수
- 부가 기능 패시브 5~6개 모두 활성
- Max HP 최대치
- 임시수리 자원 풍부

### 부가 기능 (패시브) 후보

시작 시 모두 활성. 손상 누적에 따라 하나씩 비활성:
- 핸드 사이즈 +1
- 턴 시작 시 쉴드 +3
- 연속 카드 사용 시 데미지 보너스
- 카드 1회 리드로우
- 광학 보정 (크리티컬 +20%)
- 전투 시작 시 카드 1장 추가 드로우

## 7. 임시수리 = 부채 시스템

### 핵심 메커닉

**임시수리는 무료가 아니라 미래에서 빌려오는 빚**.

- 거점에서 임시수리 시 → **빚 카운터** +1
- 보스 클리어 시 빚 카운터에 비례한 페널티 자동 적용

| 빚 카운터 | 보스 후 페널티 |
|---|---|
| 0 | 없음 |
| 1~2 | Max HP -2 추가 감소 |
| 3~4 | Max HP -2 + 부가 패시브 1개 추가 비활성 |
| 5+ | Max HP -3 + 패시브 1개 + 카드 1장 강제 손상 |

플레이어는 매 막마다 *"이번 막에서 임시수리를 몇 번 쓸 것인가"* 결정. 빚이 누적되면 후반이 가속도로 무너짐.

### 빚의 가시성
- 명시적으로 표시 (UI에 빚 카운터 숫자)
- 동시에 캐릭터 손상 CG가 빚이 많을수록 빠르게 진행 (이중 시각화)

### 자폭 강제 분기
빚 카운터가 임계값(예: 8 이상)을 넘으면 최종 거점에서 자폭만 가능. 그 미만이면 양쪽 선택 가능.

## 8. 막 구조 — 안전한 곳으로의 도주

여정 = *적 영역에서 멀어져 안전한 낙원으로 향함*. 따라서 후반으로 갈수록 적의 밀도와 위협이 떨어지지만, 플레이어 본인도 더 약해져 있어 *상대적 난이도는 유지*되거나 오히려 증가.

| 구간 | 위치 | 적 위협 | 플레이어 강도 |
|---|---|---|---|
| 1막 | 적 영역 한복판, 최전선 | ★★★★ | ★★★★★ |
| **1막 보스** | 영역 경계 수호자 | ★★★★★ | ★★★★ |
| 2막 | 침식된 도시, 황야 | ★★★ | ★★★ |
| **2막 보스** | 떠도는 거물 | ★★★★ | ★★★ |
| 3막 | 미개척 폐허 | ★★ | ★★ |
| **3막 보스** | 길 잃은 잔당 | ★★★ | ★★ |
| 최종 거점 | 낙원 입구 | — | (자폭/돌파 분기) |
| **최종 보스** | 낙원 직전 | ★★★★★ (≒1막 보스) | ★ ~ ★★ |

### 거점 시스템
- 각 막 안에 거점 1~3개
- 거점에서 가능한 행동:
  - **점검** (덱 확인, 패시브 확인, 다음 적 정보)
  - **임시수리** (현재 HP +N, 빚 +1)
  - 나중 추가 가능: 일회성 이벤트 / 작은 회복 / 정보 입수 등

### 거점 사이 게임플레이
- StS 스타일 *분기 노드 맵*. 전투 / 엘리트 / 거점 / 이벤트 노드
- 플레이어가 경로를 선택하면서 *어떤 손실을 감수할지* 매 분기 결정

### 최종 보스 — 1막 보스와 같은 종

이것이 게임의 *결정적 서사 장치*.

- 같은 *종*의 다른 개체 (또는 살아남은 1막 보스 자신)가 여정 내내 추적해 와 낙원 직전에 마주침
- 두 가지 해석 가능:
  - (a) 1막에서 죽인 보스의 *동족*이 복수를 위해 추적
  - (b) 1막 보스 자체가 살아남아 끝까지 따라옴 (개체 일치)
- 두 해석 모두 *플레이어가 1막 보스를 죽인 행위가 최종 보스를 만들었을지도 모른다*는 가능성을 남김
- 두 번째 플레이부터 플레이어는 *1막 보스를 마주칠 때 끝을 떠올리며* 임함 — 메타 인지가 추가됨

## 9. 적 디자인 — 이계종

### 톤
- 곤충형 키틴 + 차원 균열의 비현실 형태 결합
- 군집 + 모체 구조
- 색감: 보라/녹/검정 위주 (시안 톤의 로봇/UI와 강한 대비)

### 적 종류 (작명 임시)
- **일꾼 개체** — 균열에서 나온 작은 것들. 잡몹
- **사냥개 종** — 빠른 추적자
- **파수꾼 종** — 영역 보스급 (1막 보스, 최종 보스의 종)
- **떠돌이 거물** — 2막 보스
- **잔당 / 길 잃은 변종** — 3막 보스

### 핵심: 1막 보스와 최종 보스의 시각적 일관성
- 동일한 종 → 동일한 디자인 베이스
- 차이는 크기, 상처, 추적의 흔적 등으로 표현
- 플레이어가 *마지막에 만났을 때 즉시 알아볼 수 있게*

## 10. 비주얼 — 상태별 CG

### 얼굴 / 몸 분리 원칙
- 얼굴은 일러스트가 대체로 고정 (머리카락 애니메이션 영역 보호)
- 머리카락과 겹치지 않는 얼굴 요소는 수정 가능: 눈, 뺨, 턱, 입
- **몸(body) CG만 상태별로 제작**

### 몸 상태 단계

4단계, HP % 기준 자동 전환:
- **정상** (100%~75%): 현재 타이틀 일러스트
- **손상** (75%~50%): 스크래치, 작은 균열
- **심각** (50%~25%): 외장 패널 파손, 스파크
- **임계** (25%~): 팔/다리 손실, 노출된 회로

### 눈 상태 (셰이더로 처리 가능)
- 정상: 시안 양쪽 밝게
- 손상: 채도 낮아짐
- 심각: 한쪽 눈 흐릿 / 깜빡임
- 임계: 한쪽 꺼짐 or 양쪽 붉은색 (에러 모드)

별도 PNG 없이 hue shift + brightness 셰이더로 구현 가능.

### 빚 카운터의 시각화
- 빚이 많을수록 손상 CG의 단계가 *HP 비율보다 빠르게* 진행됨
- 즉 같은 HP 70%여도 빚 0인 캐릭터와 빚 5인 캐릭터의 외형이 다름

### 타이틀 화면 동적 반영
현재 런의 손상 상태에 따라 타이틀 화면의 로봇이 해당 상태 CG로 표시. 플레이 도중 타이틀로 돌아오면 *지금 얼마나 부서졌는지가 보이는* 구조.

## 11. 도련님의 대사 전달 방식

- **텍스트 only + 효과음** (성우 X)
- HUD 내 무선 통신 UI. 라디오 지직거림 SFX
- 거점에서 짧은 격려 / 정보 제공
- 중요 전투 중: 1~2줄의 한마디 삽입
- 엔딩에서만 시각화 (도련님 CG 1~2장)

**톤**:
- 3인칭 내레이션 금지. 직접 다이얼로그
- 애정은 드러내되 과하지 않음. *"수호기사"라는 호칭으로 거리감과 존경을 동시에 유지*
- 평생 함께한 사이의 *익숙한 농담 + 잔잔한 의지*
