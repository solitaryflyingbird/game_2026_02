# 다중 맵 — 사이클 분해 + 테스트 케이스

**날짜:** 2026-05-03
**선행 spec:** `다중 맵 — 자료구조 설계.pdf`
**적용 방법론:** 코딩 과업 관리 실천 방법론

---

## §0. 방법론 적용 결

### 0-1. 인용 방법론

> 1. 크게 쪼개고
> 2. 쪼갠 각각을 테스트로 검증 가능한 기능 단위까지 다시 쪼갠 후
> 3. 3~7개 단위로 투두에 올려서 치우고 체크하라

> 사이클 = ① 구현 단위 정의 → ② 구현 → ③ 디버깅 장치 통과 → ④ 테스트 통과 → ⑤ 체크 & 다음

> 쪼개기 기준 = 시간 X, **구조의 완성** O. "이 단위가 테스트로 검증 가능한가" 가 충분히 작은지 판단 기준.

### 0-2. 본 차 적용

다중 맵 도입 = 큰 과업.

- **1단계 (크게)**: M-1 ~ M-4 (4 큰 단위)
- **2단계 (기능 단위)**: 각 M 을 testable unit 으로 다시 쪼갬 (총 ~ 22 unit)
- **3단계 (3~7 개 묶음)**: 각 사이클에 5~7 개 unit 넣어 체크

각 unit 에는:
- **구현 단위 정의** — 무엇을 만들 것인가 (한 줄)
- **완료 조건** — 테스트 결로 명시 (자동 검증 가능 결로)
- **디버깅 장치** — 사이클 안 검증 결 (push_warning / 콘솔 / 헤드리스)
- **통과 기준** — 모든 완료 조건 PASS

## §7. 사이클 안 검증 흐름 (방법론 ② → ④)

### 7-1. 매 unit 작업 흐름

1. **구현 단위 정의** (위 §2~5 의 표) 읽기
2. **완료 조건** 명시 받기 (이미 표에 박힘)
3. **구현** — 코드 작성
4. **디버깅 장치 통과**:
   - GDScript `push_warning` / `assert`
   - 콘솔 출력 (LimboConsole)
   - 헤드리스 검증 스크립트 PASS
5. **테스트 통과** — 사전 정의 완료 조건 모두 PASS
6. **체크** — TaskUpdate 결로 완료 표시 → 다음 unit

### 7-2. 사이클 안 단위 우선 순위

같은 사이클 안 unit 들은 **자료구조 → 함수 → UI** 순으로 진행 (자료구조 상향식 결).

### 7-3. 사이클 종료 시점

사이클의 모든 unit PASS + 회귀 6 PASS 유지 → 다음 사이클 진입.

회귀가 깨지면 그 자리에서 정지. 다음 진행 X.

## §9. 디버깅 장치 (방법론 ③)

### 9-1. 프레임워크 레벨 검증

- **`assert(...)`** — 자료구조 정합 검증 (예: `assert(MAPS.has(STARTING_MAP))`)
- **`push_warning(...)`** — 비정상 결 알림 (예: 알 수 없는 target_map / 슬롯 결 어긋남)
- **`LimboConsole`** — 런타임 디버그 (이미 박혀있음 — `show_run` / `move` / `explore` / `rest`)
- **헤드리스 자동 검증** — 매 사이클 종료 시 PASS 검증 (= 방법론 ④)

### 9-2. 신규 콘솔 명령 후보

- `move_to_map <map_id>` — 디버그 결 즉시 이동 (개발 결, 본 차로는 보류 가능)
- `show_map` — 현 맵 / visited / explored 덤프

## §10. 진행 의지 / 미해결

### 10-1. 사이클 단위 진행 결
각 사이클 종료 시 사용자 합의 한 줄 (체크 OK / 다음 진입). 또는 M-1 ~ M-4 일괄 진행 후 PR 묶음.

### 10-2. 미해결
- (a) M-1 의 사이클 5 (UI 호환) 까지 한번에 갈지 / 사이클 1~4 만 진행 후 UI 별도 차로 갈지?
- (b) ruin_01 의 크기 (10×8 / 16×12 / 다른 결) — 본 차에는 컨텐츠 비중 0, schema 검증용이라 작은 결 충분
- (c) 사이클 묶음 (12 사이클) 을 별도 커밋으로 갈지 / M 단위 (4 커밋) 로 묶을지
- (d) 후속 연구 — "큰 기능에서 작은 기능 쪼개기" 방법 결 (방법론 마지막 §) 은 본 차의 결정 결로 박지 않음. 별도 차에서 탐구.
