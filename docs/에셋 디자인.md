# LAST MISSION — 에셋 후보 목록

> 2026-04-12 작성. 게임 화면 / 시스템 영역 전반에서 필요할 수 있는 비주얼 에셋을 가능한 한 많이 나열. 사용자가 직접 가지치기하는 용도. 타이틀 화면 에셋(완료)은 제외.
>
> 표기:
> - **★** = 거의 확실히 필요
> - **○** = 있으면 좋음
> - **△** = 선택적 / 셰이더/코드로 대체 가능

---

## 1. 캐릭터 — 주인공 (호위 안드로이드)

### 1.1 몸 상태 CG (전신, 정면)
- ★ 정상 (100%~75%) — 현재 일러스트
- ★ 손상 (75%~50%) — 작은 균열, 스크래치
- ★ 심각 (50%~25%) — 외장 파손, 노출 회로, 스파크
- ★ 임계 (25%~) — 팔/다리 손실 또는 늘어짐, 큰 손상
- △ 자폭 발동 직전 — 빛이 모이는 한 컷
- △ 자폭 직후 (잿더미 / 녹은 잔해) — 엔딩 B 전용

### 1.2 아이들 애니메이션 (각 상태별)
- ★ 정상 8프레임 (현재 보유)
- ○ 손상 8프레임 (덜 움직임, 살짝 떨림)
- ○ 심각 8프레임 (한쪽으로 기우뚱)
- △ 임계 8프레임 (거의 정지, 작은 떨림만)
- *대안*: 손상 단계는 정지 이미지 + 미세한 셰이더 떨림으로 대체 가능

### 1.3 표정 / 눈 상태
- ○ 눈 라이트 정상 (시안 양쪽)
- ○ 눈 라이트 약함 (낮은 채도)
- ○ 한쪽 눈 꺼짐
- ○ 양쪽 눈 붉은색 (에러 모드)
- △ 셰이더로 hue/brightness만 조절하면 별도 PNG 불필요

### 1.4 전투 액션 포즈 (텍스트 모드면 생략 가능)
- △ ATK 발동 — 공격 자세
- △ BLK 발동 — 방어 자세
- △ POWER 발동 — 차징/포효 자세
- △ 피격 — 움찔/뒤로 밀림
- △ 사망 (쓰러짐)
- △ 승리 (서있음)

### 1.5 대화/컷씬용 흉상 (반신 컷)
- ○ 정상 흉상
- ○ 손상 흉상
- ○ 심각 흉상
- ○ 임계 흉상
- *용도*: 대사 박스 옆 portrait

## 2. 캐릭터 — 도련님 (주인)

게임 본편에는 등장하지 않음. 무선 통신은 텍스트만으로 처리. 엔딩 CG만 한 장씩.

- ★ 생환 엔딩 CG 1장 (도련님 + 로봇 함께 있는 컷)
- ★ 자폭 엔딩 CG 1장 (도련님 혼자 낙원 도착)

## 4. 카드

### 4.1 카드 프레임 / 배경
- ★ ATTACK 타입 프레임
- ★ BLOCK 타입 프레임
- ★ POWER 타입 프레임 (지속 효과)
- ○ SPECIAL 타입 프레임
- ○ 등급별 테두리 (일반/희귀/엘리트/유물)

### 4.2 카드 일러스트 (1장당 1개)

**핵심 작업량 — 카드 풀 전체를 그림으로 채워야 함.**

- ★ 카드 일러스트 베이스 ~30종 (시작 풀, 코어 카드)
- ★ 추가 카드 일러스트 ~30~50종 (보상으로 얻거나 발견하는 확장 카드)
- ○ 등급/강화 변형 (강화 카드는 살짝 다르게: 색감 변화 / 디테일 추가 / 이펙트 오버레이)

**카드 일러스트 스타일 가이드 (권장)**:
- 통일된 화풍 (화면 일관성 위해)
- 정사각형 또는 4:3 비율
- 카드 프레임 안에 들어가는 클로즈업 / 액션 컷
- 톤은 시안 + 화이트 액센트 (UI 톤과 일치)
- 너무 디테일하지 않게 — 작게 표시되니 실루엣과 핵심 색감으로 인지 가능해야 함

**카드 종류별 일러스트 톤 제안**:
- **공격 (ATTACK)** — 검 / 빔 / 펀치 / 발사 / 슬래시 등 액션 모먼트
- **방어 (BLOCK)** — 쉴드 / 자세 / 차단 / 거치 무기 등 정적 자세
- **파워 (POWER, 지속)** — 부스트 / 코어 빛남 / 충전 / 시스템 활성화 등
- **스페셜 (특수)** — 자폭 / 오버드라이브 / 기술명 카드 등 강력한 일회성

**대안**:
- *최소 버전*: 카드별 실제 일러스트 없이 *아이콘 + 추상 이미지*만으로 시작 (예: ATTACK 카드는 모두 검 아이콘 베이스 + 색만 변화)
- *중간 버전*: 카드 타입별 5~10가지 *공통 일러스트 풀*을 만들고 카드들이 공유 (Slay the Spire의 이전 버전 방식)
- *풀 버전*: 카드 1장당 1개의 고유 일러스트 (이상적, 가장 큰 작업)
- 개발 단계에서는 텍스트 라벨만으로 충분 — 일러스트는 후반 폴리시에 입힘

### 4.3 카드 컴포넌트
- ★ 코스트 오브 (좌상단) — 숫자 1~3, X
- ★ 카드 이름 배너
- ★ 효과 텍스트 박스
- ○ 타입 라벨 (ATTACK/BLOCK/POWER)
- ○ 타겟 표시 아이콘 (단일/전체/자기)

### 4.4 카드 상태
- ○ 호버 (살짝 위로)
- ○ 선택 (강조)
- ○ 비활성/사용 불가 (회색)
- ○ 손상된 카드 (균열 텍스처)
- △ 셰이더 변조로 대부분 대체 가능

### 4.5 카드 백 (뒷면)
- ○ 덱 표시용 뒷면 1종

### 4.6 카드 애니메이션
- ○ 드로우 모션 (덱 → 핸드)
- ○ 플레이 모션 (핸드 → 적 / 자기)
- ○ 버린덱 행 모션
