# 데이터 설계 — JSON 전역 데이터

> 카드 시스템 전환에 필요한 최소 데이터만 정의. 확장은 기본 루프가 돈 후에.

---

## 즉시 필요 (Phase A~F)

### `data/cards.json`

```json
{
  "atk_basic": {
    "name": "기본 공격",
    "type": "ATTACK",
    "cost": 1,
    "damage": 6,
    "block": 0,
    "desc": "적에게 6 데미지."
  },
  "blk_basic": {
    "name": "기본 방어",
    "type": "BLOCK",
    "cost": 1,
    "damage": 0,
    "block": 5,
    "desc": "방어력 5 획득."
  }
}
```

### `data/enemies.json`

```json
{
  "test_dummy": {
    "name": "테스트 더미",
    "max_hp": 20,
    "actions": [
      {"kind": "attack", "value": 5}
    ]
  }
}
```

### `data/config.json`

```json
{
  "starting_deck": [
    "atk_basic", "atk_basic", "atk_basic", "atk_basic", "atk_basic",
    "blk_basic", "blk_basic", "blk_basic", "blk_basic", "blk_basic"
  ],
  "starting_hp": 50,
  "starting_max_hp": 50,
  "energy_per_turn": 3,
  "draw_per_turn": 5
}
```

## 나중에 추가 (기본 루프 동작 이후)

| 파일 | 내용 | 시점 |
|---|---|---|
| cards.json 확장 | 공격/방어 외 POWER, SPECIAL 카드 추가 | Phase G (보상 시스템) |
| enemies.json 확장 | 잡몹 2~3종 + 보스 4종 | Phase G 이후 |
| `data/encounters.json` | 막별 조우 테이블 | 노드 맵 구현 시 |
| `data/passives.json` | 부가 기능 정의 | 패시브 시스템 구현 시 |
| `data/statuses.json` | 상태이상 (약화/취약/강화) | 상태이상 구현 시 |
| config.json 확장 | 빚 페널티, max_hp 감소 곡선 | 빚 시스템 구현 시 |

---

## 로딩 방식

`GameData._ready()`에서:
```gdscript
var cards_file = FileAccess.open("res://data/cards.json", FileAccess.READ)
var cards = JSON.parse_string(cards_file.get_as_text())
```

코드(`*.gd`)와 데이터(`*.json`)는 폴더 분리: `res://data/`.
