# 배틀시스템 명세

> 구현 전 확정된 설계. 이 문서에 없는 것은 만들지 않는다.

---

## 1. 데이터 소유권

```
GameData (const)          카드 템플릿 정의. 불변.
    ↓ 복제
RunManager (run_data)     덱 원본 소유. 런 전체 수명.
    ↓ 복사본 전달
BattleManager             전투 중만 존재. 덱 빌려 쓰고 결과만 반환.
```

- **GameData** → 카드 정의 (`CARDS` dict). 절대 안 바뀜.
- **RunManager** → `run_data["deck"]` = 카드 인스턴스 배열. 런 시작 시 GameData에서 복제. **덱 영구 변동 권한은 RunManager에만 있음.**
- **BattleManager** → 전투 시작 시 덱 복사본 받음. 전투 중 draw/hand/discard/exhaust 조작. 전투 끝나면 `{outcome, hp}` 만 반환. **덱 자체를 돌려주지 않음.**

## 2. 카드 데이터 구조

### 2.1 카드 템플릿 (GameData — 불변)

```gdscript
const CARDS = {
    "atk_basic": {
        "name": "기본 공격",
        "type": "ATTACK",
        "cost": 1,
        "damage": 6,
        "block": 0,
        "desc": "적에게 6 데미지.",
    },
    "blk_basic": {
        "name": "기본 방어",
        "type": "BLOCK",
        "cost": 1,
        "damage": 0,
        "block": 5,
        "desc": "방어력 5 획득.",
    },
}
```

### 2.2 카드 인스턴스 (덱에 들어가는 실제 카드 1장)

```gdscript
{
    "id": "atk_basic",      # 어떤 카드인지 (GameData.CARDS 키)
    "upgraded": false,       # 강화 여부
    "damaged": false,        # 손상 여부
}
```

- 같은 `"atk_basic"` 5장이어도 **5개의 독립 dictionary**
- 1장만 강화하면 그 dict만 `upgraded: true`
- 실제 스탯은 `GameData.CARDS[inst.id]` 에서 기본값 조회 → `upgraded`/`damaged` 에 따라 보정

### 2.3 스탯 보정 규칙 (추후 튜닝)

```
최종 damage = base_damage + (upgraded ? +3 : 0) + (damaged ? -2 : 0)
최종 block  = base_block  + (upgraded ? +3 : 0) + (damaged ? -2 : 0)
최종 cost   = base_cost   + (damaged  ? +1 : 0)
```

구체 수치는 나중에. 구조만 확정.

## 3. 덱 구조

### 3.1 RunManager 레벨 (런 전체)

```gdscript
run_data["deck"] = [
    {"id": "atk_basic", "upgraded": false, "damaged": false},
    {"id": "atk_basic", "upgraded": false, "damaged": false},
    {"id": "atk_basic", "upgraded": false, "damaged": false},
    {"id": "atk_basic", "upgraded": false, "damaged": false},
    {"id": "atk_basic", "upgraded": false, "damaged": false},
    {"id": "blk_basic", "upgraded": false, "damaged": false},
    {"id": "blk_basic", "upgraded": false, "damaged": false},
    {"id": "blk_basic", "upgraded": false, "damaged": false},
    {"id": "blk_basic", "upgraded": false, "damaged": false},
    {"id": "blk_basic", "upgraded": false, "damaged": false},
]
```

### 3.2 BattleManager 레벨 (전투 중)

```gdscript
draw_pile:    [카드인스턴스...]   # 아직 안 뽑은 카드
hand:         [카드인스턴스...]   # 현재 손패 (최대 5~6장)
discard_pile: [카드인스턴스...]   # 사용/버린 카드 (리셔플로 돌아옴)
exhaust_pile: [카드인스턴스...]   # 이번 전투에서 소멸 (전투 끝나면 버림)
```

### 3.3 전투 시작 시 초기화

```
1. RunManager.run_data["deck"] 을 deep copy → BattleManager.draw_pile
2. hand = []
3. discard_pile = []
4. exhaust_pile = []
5. draw_pile 셔플
```

### 3.4 전투 종료 시

```
1. draw_pile, hand, discard_pile, exhaust_pile 전부 폐기
2. RunManager에 { outcome: "win"/"lose", hp: 잔여HP } 만 반환
3. RunManager의 원본 덱은 그대로 유지 (전투 중 변동 없음)
```

## 4. 전투 참여자 데이터

### 4.1 플레이어 (RunManager에서 가져옴)

```gdscript
# BattleManager가 참조하는 플레이어 상태
player = {
    "hp": run_data["hp"],           # 현재 HP (전투 중 변동, 전투 후 RunManager에 반환)
    "max_hp": run_data["max_hp"],   # 최대 HP
    "block": 0,                      # 방어막 (턴 시작 시 리셋)
    "energy": 3,                     # 현재 에너지 (턴 시작 시 충전)
    "energy_max": 3,                 # 턴당 최대 에너지
}
```

### 4.2 적 (최대 ~5마리, 배열)

```gdscript
enemies = [
    {
        "id": "test_dummy",
        "name": "테스트 더미",
        "hp": 20,
        "max_hp": 20,
        "block": 0,                  # 적도 방어막 있음, 턴 시작 시 리셋
        "intent": {},                # 이번 턴 예고 행동
```

## 5. 턴 흐름

### 5.1 턴 시작 (TURN_START)

```
1. player.block = 0
2. 각 enemy.block = 0
3. player.energy = player.energy_max
4. draw(5)  → draw_pile에서 hand로 5장 이동
   - draw_pile 부족 시: discard_pile → draw_pile로 합침 + 셔플
5. 각 적의 intent 결정 (actions에서 랜덤 또는 패턴)
```

### 5.2 플레이어 턴 (PLAYER_TURN)

플레이어가 반복적으로 카드를 사용하는 루프.

```
반복:
  1. 플레이어가 hand에서 카드 클릭
  2. energy >= card.cost 확인
     - 부족하면 → "에너지 부족" 피드백, 무시
  3. 타겟 결정
     - ATTACK + 적 2마리 이상 → 타겟 선택 대기
     - ATTACK + 적 1마리 → 자동 타겟
     - BLOCK → 타겟 = 자기 (선택 불필요)
  4. 효과 적용
     - damage > 0: target.block 먼저 흡수, 남은 건 target.hp 감소
     - block > 0: player.block += block
  5. energy -= card.cost
  6. 사용된 카드: hand → discard_pile (또는 exhaust 속성이면 exhaust_pile)
  7. 적 전멸 확인 → 모든 적 hp ≤ 0이면 COMBAT_WIN
  8. 아니면 → 다음 카드 대기 (또는 턴 종료 클릭)
```

### 5.3 턴 종료 (TURN_END)

```
1. hand에 남은 카드 전부 → discard_pile
2. (추후: 턴 종료 시 효과 처리)
```

### 5.4 적 턴 (ENEMY_TURN)

```
1. 살아있는 적 각각:
   a. intent 읽기
   b. kind == "attack":
      - incoming = intent.value
      - absorbed = min(player.block, incoming)
      - player.block -= absorbed
      - player.hp -= (incoming - absorbed)
   c. (추후: kind == "block" → enemy.block += value)
   d. (추후: kind == "buff" → 적 강화)
2. player.hp ≤ 0 → COMBAT_LOSE
3. 아니면 → TURN_START (다음 턴)
```

## 6. 덱 조작 함수 목록

이 6개가 덱 관련 함수의 전부. 이 외의 덱 조작은 없음.

| 함수 | 동작 | 호출 시점 |
|---|---|---|
| `shuffle(pile)` | pile을 제자리에서 섞음 | 전투 시작, 리셔플 |
| `draw(n)` | draw_pile에서 n장 → hand. 부족하면 reshuffle 먼저 | 턴 시작 |
| `play_card(index, target)` | hand에서 1장 제거 → discard_pile. 효과 적용 | 플레이어 카드 사용 |
| `exhaust_card(index)` | hand에서 1장 제거 → exhaust_pile | 소멸 카드 사용 시 |
| `discard_hand()` | hand 전부 → discard_pile | 턴 종료 |
| `reshuffle()` | discard_pile 전부 → draw_pile에 합침 → 셔플 → discard 비움 | draw 부족 시 |

## 7. 데미지 계산

```
# 공격 → 대상에게
incoming = card.damage (+ 보정)
absorbed = min(target.block, incoming)
target.block -= absorbed
target.hp -= (incoming - absorbed)

# 방어 → 자기에게
player.block += card.block (+ 보정)
```

- 블록은 **턴 시작 시 0으로 리셋** (플레이어, 적 모두)
- 오버블록은 낭비 (블록 10, 데미지 5 → 블록 5 소모, 나머지 5는 턴 끝에 사라짐)

## 8. 전투 결과 반환

```gdscript
# BattleManager → RunManager
{
    "outcome": "win",   # 또는 "lose"
    "hp": 35,           # 전투 후 잔여 HP
}
```

이것만 반환. 덱 상태, 적 상태, exhaust 목록 등은 반환하지 않음.
RunManager의 원본 덱은 전투 전후로 동일.

---

## 9. 덱 영구 변동 (전투 밖 — RunManager 관할)

전투 중에는 절대 안 일어남. 다음 시점에만:

| 시점 | 변동 | 예시 |
|---|---|---|
| 에피소드(막) 전환 | 카드 영구 제거 | "덱에서 카드 1장 선택해서 제거" |
| 에피소드 전환 | 방해카드 삽입 | 저주 카드, 손상 카드 덱에 추가 |
| 에피소드 전환 | 카드 손상 | 특정 카드의 `damaged = true` |
| 보상 (추후) | 카드 획득 | 새 카드 인스턴스 덱에 추가 |
| 거점 (추후) | 카드 강화 | 특정 카드의 `upgraded = true` |
| 빚 페널티 (추후) | 카드 손상/제거 | 빚 카운터에 따른 강제 손상 |

## 10. 이 명세에 없는 것 (지금 만들지 않음)

- 상태이상 (약화/취약/강화)
- 패시브 시스템
- 노드 맵
- 보상 카드 선택 (3장 택1)
- 거점/임시수리
- 빚 카운터
- 자폭 분기
- 카드 풀 확장 (2종 외)
- 적 다양화 (test_dummy 외)
- UI 비주얼 (텍스트 버튼만)

위 항목들은 이 명세의 전투 루프가 **동작 확인된 후에** 추가.
