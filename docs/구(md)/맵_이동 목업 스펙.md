# 맵 이동 · 식량 목업 스펙

> **대상**: 클로드 웹 아티팩트 (React 단일 파일).
> **목적**: 노드 그래프 맵 이동과 식량 자원 소비를 브라우저에서 바로 시연.
> **제외**: 전투·캐릭터·카드 등 본 게임의 다른 모든 시스템.

---

## 1. 목표

- 데이터 기반 노드 그래프 맵 생성 (5 레이어, 분기 있음)
- 플레이어가 인접 노드로 이동 (전진·후진 모두 가능)
- 이동 1회당 식량 1 소모
- `rest` 노드 방문 시 식량 회복
- 식량 고갈 → 게임오버
- 최종 레이어 노드 도달 → 승리

---

## 2. 자료구조

### MapNode
```
{
    id:          number,
    type:        "combat" | "rest",
    layer:       number,          // 1..5
    connections: number[],        // 양방향 엣지 (인접 노드 id)
    visited:     boolean,
}
```

### MapTemplate (생성 입력)
```
{
    layers: [
        { nodes: [ {type}, {type}, ... ] },
        ...
    ]
}
```

### GameState
```
{
    map:           { [id]: MapNode },
    currentNodeId: number,
    food:          number,
    foodMax:       number,
    phase:         "playing" | "lose" | "victory",
}
```

---

## 3. 맵 템플릿 (고정)

```js
{
    layers: [
        { nodes: [{ type: "combat" }] },                                   // L1: 1개
        { nodes: [{ type: "combat" }, { type: "rest" }] },                 // L2: 2개
        { nodes: [{ type: "combat" }, { type: "combat" }] },               // L3: 2개
        { nodes: [{ type: "combat" }, { type: "rest" }] },                 // L4: 2개
        { nodes: [{ type: "combat" }] },                                   // L5: 1개 (최종)
    ]
}
```

---

## 4. 맵 생성 알고리즘

1. 각 노드에 `id` 순차 부여 (1부터), `layer` 기록
2. 각 노드의 `visited = false` 로 초기화
3. 레이어 간 연결:
   - 레이어 N 의 각 노드 i 는 레이어 N+1 의 노드 i 에 연결 (인덱스 매칭, 크기 초과 시 clamp)
   - 추가: 노드 i 가 레이어 N+1 의 노드 i+1 이 존재하면 그쪽에도 연결 (교차 엣지)
4. **양방향 처리**: `A→B` 만들 때 `B→A` 도 자동 추가
5. 반환: `Dictionary<id, MapNode>`

**결과 예 (위 템플릿 기준)**:
```
L1:   [1]
L2:   [2, 3]
L3:   [4, 5]
L4:   [6, 7]
L5:   [8]

연결:
1 ↔ 2, 1 ↔ 3
2 ↔ 4, 2 ↔ 5
3 ↔ 5
4 ↔ 6, 4 ↔ 7
5 ↔ 7
6 ↔ 8
7 ↔ 8
```

---

## 5. 이동 규칙

`moveTo(id)`:

1. `currentNode.connections` 에 `id` 없으면 무시 (invalid move)
2. `food -= 1`
3. `currentNodeId = id`, `map[id].visited = true`
4. 식량 고갈 체크: `food <= 0` → `food = 0`, `phase = "lose"`, 종료
5. 방문 노드가 **rest 타입** 이면 `food = min(food + 10, foodMax)`
6. 방문 노드가 **최종 레이어(L5)** 이면 `phase = "victory"`

> 최종 노드는 `type: "combat"` 이지만 layer 기준으로 승리 판정.

---

## 6. 초기값

| 항목 | 값 |
|---|---|
| 시작 노드 | 레이어 1 유일 노드 (`id = 1`) |
| `food` | 20 |
| `foodMax` | 20 |
| 이동 소모 | 1 |
| rest 회복 | +10 |

시작 시 `map[1].visited = true`, `currentNodeId = 1`.

## 7. UI 요구사항

### 레이아웃

- **상단 바**:
  - 식량 `🍞 15 / 20` (저식량 시 붉게)
  - 현재 위치 `L2 · Node 2`
  - 재시작 버튼

- **중앙 — 맵 시각화**:
  - 레이어별 **세로 스택** (L1 하단 → L5 상단 또는 좌→우, 취향)
  - 각 레이어의 노드는 수평 배치
  - 노드 표기:
    - 원 또는 둥근 사각 타일
    - 타입별 아이콘: `combat = ⚔️`, `rest = 🏠`
    - 최종 노드는 특별히 강조 (왕관 `👑` 등)
    - `visited` 은 탁한 색, 미방문은 밝은 색
    - `currentNodeId` 는 테두리·glow 로 하이라이트
  - `connections` 를 SVG 선으로 그리기 (노드 중심 연결)

- **하단 — 이동 패널**:
  - 현재 노드의 connections 에 있는 노드들이 버튼으로 나옴
  - 각 버튼에 `[노드 id] L{layer} {type_icon} {방문 여부}` 표시
  - 클릭 → `moveTo(id)`

### 상호작용

- 이동 가능 노드 **하이라이트** (펄스 애니메이션 등)
- 이동 시 current 표시 스무스 전환
- 식량 감소 시 숫자에 짧은 shake/flash
- rest 에서 식량 회복 시 초록 flash

### 종료 화면

- `phase === "lose"`: "보급 소진 — 실패" 모달 + 재시작
- `phase === "victory"`: "최종 노드 도달 — 승리" 모달 + 재시작

## 8. 스코프 명시 (만들지 말 것)

- 전투 시스템 (combat 노드 방문은 이동 그 자체. 아무 상호작용 없음.)
- 히로인 · 양팔 · HP · 카드
- 랜덤 맵 생성 (고정 템플릿만)
- 저장 / 로드
- 사운드

---

## 9. 구현 힌트

- **단일 React 컴포넌트** 로 충분
- 상태: `useState<GameState>` 하나
- 맵 시각화는 **SVG** 권장 (노드는 `<circle>` + `<text>`, 엣지는 `<line>`)
- 노드 좌표는 `layer` 와 레이어 내 `index` 로부터 유도 (예: `x = layer * 140`, `y = 60 + index * 80`)
- 재시작은 초기 GameState 를 다시 만들어 setState
- Tailwind 있으면 사용 (없으면 inline style)
- 키보드 단축키(선택): 숫자키로 이동 가능 노드 선택

## 10. 검증 시나리오

목업이 제대로 동작하는지 확인할 플레이 흐름:

1. 시작 → L1 노드에서 L2 의 combat 노드로 이동 → 식량 19
2. L2 → L2 rest 노드로 후진(??) — 아니, **L2 내부 이동은 불가**. L1→L2(combat)→L3 진행
3. 경로 중 rest 방문 시 식량 +10 확인
4. 일부러 20 번 이상 왔다갔다 → 식량 0 → lose 모달
5. 곧장 L5 최종 노드 도달 → victory 모달
6. 재시작 → 초기 상태 (food 20, L1 currentNode)
